# 연구 결과: 오늘의 할 일 웹 앱

## 결정 사항

- **프런트엔드 렌더링 방식**: 단일 HTML 템플릿과 바닐라 JavaScript를 사용한다. 서버는 HTML을 제공하고, 화면 갱신은 fetch API로 REST API를 호출해 처리한다.
- **데이터 저장 방식**: SQLite를 사용하고, 데이터베이스 경로는 .env의 DATABASE_URL로 관리한다.
- **API 설계 방식**: REST 스타일의 JSON API를 사용하되, 메인 화면은 HTML 템플릿으로 제공한다.
- **테스트 전략**: API 수준의 동작 검증은 pytest와 FastAPI TestClient를 사용한다.

## 근거

- 요구사항에 명시된 기능은 CRUD 중심으로 구현 범위가 명확하다.
- SQLite는 로컬 실행 환경에서 추가 설정 없이 바로 사용 가능하다.
- 단일 페이지 구조와 별도 빌드 도구가 없는 조건에서 Jinja2 + 바닐라 JavaScript 조합이 가장 단순하다.
- FastAPI와 Pydantic 조합은 입력 검증과 테스트 작성이 간단하다.

## 검토한 대안

- **대안 1: React/Vite**: 더 풍부한 UI 구성 가능하지만, 요구사항에 비해 과도한 도구와 설정 비용이 발생한다.
- **대안 2: Flask + Jinja2**: FastAPI보다 API 문서화와 타입 검증 측면에서 구현 편의성이 낮다.
- **대안 3: 데이터베이스 없이 메모리 저장**: 영속성 요구를 만족하지 못한다.
